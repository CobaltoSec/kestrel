# Phase p1.5 — Intel Recon (pre-engagement WebSearch)

## Objetivo
Antes de tocar la máquina, sintetizar `intel.md` con el chain probable (foothold + privesc) basado en información pública. Decide entre `MODE=guided` (retired con writeups) o `MODE=blind` (active o sin info).

**Principio "solo dirección":** vector probable + CVE + técnica, sin comandos copy-paste. Ver `shared/intel-prompt.md` para reglas.

---

## Variables de entrada (de p1)

- `MACHINE_NAME` — slug
- `MACHINE_OS` — `Linux` | `Windows`
- `MACHINE_DIFFICULTY` — `Easy` | `Medium` | `Hard`
- `MACHINE_RETIRED` — `true` | `false`
- `MACHINE_RATING` — float
- `SESSION_DIR` — todavía no existe (lo crea p2 después). Por ahora trabajamos con un staging path.

> Nota: p1.5 corre **antes** de p2-engagement-setup. El `intel.md` se escribe primero a `staging/htb-intel-<slug>.md` y p2 lo mueve al `SESSION_DIR` final.

---

## PASO 0 — Hard gate retired/active

```
if MACHINE_RETIRED == false:
    → ejecutar PASO 5 (intel.md mínimo, MODE=blind, skip WebSearch)
    → continuar a p2
else:
    → ejecutar PASO 1-4 (WebSearch + síntesis)
```

**Razón del gate:** HTB TOS prohíbe usar writeups en máquinas activas. Skip por TOS, no por capacidad.

Narrar a Nico:
```
📡 Intel recon — <MACHINE_NAME>
🔍 Status: retired (puedo buscar writeups) / active (TOS HTB → blind)
➡ <Si retired: "Lanzo WebSearch + síntesis (~3-5 min)" | Si active: "Skip WebSearch, voy a blind directo">
```

---

## PASO 1 — WebSearch agresivo (solo retired)

Lanzar **3-4 queries en paralelo** con WebSearch:

```
Q1: "<MACHINE_NAME> hackthebox writeup walkthrough"
Q2: "<MACHINE_NAME> hackthebox 0xdf"
Q3: "<MACHINE_NAME> hackthebox ippsec"
Q4: "<MACHINE_NAME> hackthebox foothold privilege escalation"
```

Si la máquina tiene nombre genérico (ej: "Cap", "Lame") agregar `htb` a las queries para filtrar ruido:
```
Q1: "<MACHINE_NAME> htb writeup"
```

Esperado: cada query retorna 5-10 URLs. Top sources típicas:
- 0xdf.gitlab.io / 0xdf.github.io (writeups detallados)
- ippsec.rocks (índice videos IppSec)
- youtube.com (videos)
- medium.com / hashnode / dev.to (writeups community)
- snowscan.io / juggernaut-sec / steflan-security (otros writers)

Filtrar resultados:
- ✅ Mantener: blog posts, GitHub gists/READMEs, IppSec links.
- ❌ Descartar: HTB pago oficial (`hackthebox.com/machine_writeups/...`).
- ❌ Descartar: foros de discusión sin contenido técnico (ej: "ayuda con esta máquina").

---

## PASO 2 — Scrape Top-3 con WebFetch

Elegir las 3 URLs con mejor pinta (preferir 0xdf, IppSec, snowscan).

Para cada URL, lanzar `WebFetch` con prompt:
```
Extrae el chain de ataque de este writeup HTB:
1. Servicios encontrados en recon (puerto + servicio + versión).
2. Vector de foothold (CVE/técnica + endpoint/parámetro).
3. Si hay crack de hash: tipo de hash + wordlist usada + password (si lo menciona textualmente).
4. Vector de privesc (técnica + binario/config relevante).
5. Gotchas o quirks no obvios mencionados.

NO copies bloques de código ni comandos exactos. Resumí en prosa técnica.
```

WebFetch puede tardar 5-10s por URL. Si una URL devuelve error 403/404, descartarla y tomar la siguiente del top-5.

Si las 3 URLs fallan → confidence=low → saltar a PASO 4 con info parcial.

---

## PASO 3 — Síntesis con anti-spoiler check

Tomar los 3 outputs de WebFetch + cualquier mención adicional de WebSearch.

**Cross-check primero:**
- Si los 3 writeups concuerdan en foothold + privesc → confidence=`high`.
- Si 1-2 concuerdan o hay discrepancias menores → confidence=`medium`.
- Si solo 1 fuente útil o discrepancias graves → confidence=`low`.

**Llenar template** de `shared/intel-prompt.md` con:
- Header: `MACHINE_NAME`, fecha ISO, confidence, sources.
- Resumen ejecutivo 3 líneas.
- Sección Foothold: vector, servicio, framework/versión, CVE, endpoint, crack requerido, wordlist sugerida.
- Sección Privesc: vector, path típico, hashes intermedios, tools.
- Gotchas / quirks.
- Wordlists & passwords reportadas (si writeups las nombran textualmente).
- Tools recomendadas (1 línea cada una, sin comandos).
- Referencias (URLs).

**Anti-spoiler check** antes de escribir (ver `intel-prompt.md` sección final). Si alguna casilla del check no está tildada, refactor.

Path: `sectors/red-team/htb/staging/htb-intel-<MACHINE_NAME>.md` (crear `staging/` si no existe).

## PASO 4 — Resumen para Nico + HITL crítico

Mostrar resumen de 5-7 líneas (NO el `intel.md` completo):

```
=== INTEL — <MACHINE_NAME> (retired) ===
Confidence: <high|medium|low>  (<N> writeups consultados)

📍 Foothold probable: <vector + framework + CVE>
   Crack esperado: <bcrypt/MD5/NTLM/none> + wordlist <name>

📍 Privesc probable: <técnica + path/binario>
   Hashes intermedios: <SHA1/SHA256/none>

⚠ Gotchas: <hasta 2 quirks importantes>

¿Cómo seguimos?
[s] usar intel — guided mode (recon dirigido + skip vuln genérico)
[b] ignorar — blind mode (pentest clásico desde cero)
[v] ver intel.md completo
```

Procesar respuesta:
- `s` → `MODE=guided`. Continuar al PASO 5.
- `b` → `MODE=blind`. El `intel.md` se conserva en staging pero p3 lo ignora. Continuar al PASO 5.
- `v` → mostrar `intel.md` completo + repetir prompt.

**Default si confidence=low:** sugerir `b` (blind) con un "[s] usar igual / [b] blind (recomendado)". El intel poco confiable puede llevar por el camino equivocado.

Cuando MODE=blind, anunciar a Nico que el fingerprinting L1 se ejecutará en p3 después de que la máquina esté spawn + VPN levantada:

```
   1. nmap -sV --top-ports 200 → classify ports/services → fingerprint.json

   3. Branch B de p3 usa fingerprint.json en lugar de vuln scan genérico

```

Añadir al state.json:
```json

```

Append a `sessions.jsonl`:
```json

```

Continuar a `p2-engagement-setup.md`. p2 mueve `staging/htb-intel-<NAME>.md` → `<SESSION_DIR>/intel.md`.

## Edge cases

**E1 — WebSearch rate-limited:** Si las 4 queries devuelven errores → fallback a `MODE=blind`, escribir `intel.md` minimal con nota "WebSearch no disponible".

**E2 — Writeups en idiomas múltiples:** preferir inglés y español. Filtrar otros idiomas a menos que sea la única fuente disponible.

**E3 — Writeups outdated (machine actualizada por HTB):** si los writeups son de hace >2 años Y la máquina muestra "patched"/"updated" en HTB, marcar confidence=`low` aunque haya 3+ writeups concordantes.

**E4 — Spoiler accidental:** si después de escribir `intel.md` Claude detecta que pasó el anti-spoiler check (ej: hay un comando completo), refactor antes de mostrar a Nico.

**E5 — Re-run sesión interrumpida (`/htb resume`):** si ya existe `intel.md` en `<SESSION_DIR>/`, skip toda esta phase. Leer `intel_path` del last-cycle.json y continuar.
